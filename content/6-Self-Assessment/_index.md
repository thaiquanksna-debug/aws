During the internship period from **17/04/2026 to 10/07/2026**, I focused on the direction of **Cloud Operations / Application Deployment Engineer**. My 12-week worklog covered the internship environment, Hugo, AWS Console, Free Tier, Billing, IAM, EC2, VPC, RDS, Auto Scaling, CloudWatch, and then moved into workshop documentation, proposal finalization, AWS diagram correction, and final project implementation.

At the beginning, I had limited hands-on experience with the AWS Console and was not used to recording evidence, errors, and validation results while doing labs. I also needed to build better habits for cost management, resource cleanup, and post-deployment checks such as security group and route table review. After the internship, I understood more clearly how to operate a basic AWS workload: launching servers, connecting databases, monitoring logs and metrics, checking health status, controlling cost, and documenting the implementation process.

Through the internship, I improved several key skill groups:

- **Deployment skills:** AWS Console, EC2, key pairs, security groups, user data, RDS integration, Auto Scaling Group, Launch Template, CloudWatch, and resource cleanup.
- **Operations:** Checking instance status, application logs, basic alarms, health checks, and connectivity issues between EC2, RDS, and the network layer.
- **Documentation:** Maintaining weekly worklogs, standardizing bilingual Hugo content, and mapping references from AWS Study Group and AWS Docs.
- **Teamwork:** Working with my teammate in the final two weeks to finalize the proposal, correct the inaccurate diagram, and implement the project in the same direction.

I still need to improve my ability to analyze architecture at a higher level and explain why each AWS service is selected. However, the internship helped me become more confident in deploying, testing, and documenting an AWS workshop/project according to practical requirements.

## Areas for improvement

- Study higher-level architecture design to explain the role of each service in the diagram.
- Practice Terraform/CloudFormation to automate deployment instead of relying mainly on AWS Console.
- Improve root-cause analysis for network, IAM permission, and database connection issues.
- Present technical flows more clearly when describing workload deployment and operation.
- Continue building the habit of recording complete evidence: screenshots, logs, metrics, alarms, and test results.
